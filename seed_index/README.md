seed_index Lambda
Bulk-loads dummy documents into AWS OpenSearch Serverless (AOSS) indexes for the Atlas Desk platform. Fully self-contained — all dummy data is inline in the handler.

Folder Contents
lambda/seed_index/
├── handler.py        # Lambda handler — SigV4 auth, inline dummy data, bulk indexing
└── requirements.txt  # Python dependencies
Supported Indexes
Index Name	Profile key
atlas-desk-documents-index	documents
atlas-desk-knowledge-index	knowledge
Both indexes use embedding vectors of exactly 1024 dimensions (knn_vector field).

Each profile maps to a different builder function with a schema-correct field set:

documents → build_documents_docs() — includes document_owner, excludes knowledge_domain and source_system
knowledge → build_knowledge_docs() — includes knowledge_domain and source_system, excludes document_owner
Embeddings are generated by fake_embedding() — deterministic synthetic floats, exactly 1024 values per record.

AOSS Data Access Policy Requirement
The Lambda execution role ARN must be included in the AOSS data access policy for the target collection with the following permissions:

[
  {
    "Rules": [
      {
        "Resource": ["index/YOUR_COLLECTION_NAME/*"],
        "Permission": [
          "aoss:ReadDocument",
          "aoss:WriteDocument",
          "aoss:DescribeIndex",
          "aoss:CreateIndex"
        ],
        "ResourceType": "index"
      }
    ],
    "Principal": ["arn:aws:iam::ACCOUNT_ID:role/YOUR_LAMBDA_EXECUTION_ROLE"]
  }
]
Without this, the Lambda will receive a 403 AuthorizationException from AOSS even if IAM permissions are correct. Both must be set.

Packaging the Lambda
Run from the lambda/seed_index/ directory:

# 1. Install dependencies into a local package/ folder
pip install -r requirements.txt -t package/

# 2. Copy handler into the package folder
cp handler.py package/

# 3. Zip the contents (not the folder itself)
cd package
zip -r ../seed_index.zip .
cd ..

# 4. Upload to Lambda
aws lambda update-function-code \
  --function-name YOUR_FUNCTION_NAME \
  --zip-file fileb://seed_index.zip
Handler reference in the Lambda console / Terraform: handler.handler

Notes
No external data files are required — all dummy records are defined inline in handler.py
Embedding dimension is exactly 1024 — AOSS will reject documents with a different vector length
last_modified_at follows the format YYYY-MM-DD HH:MM:SS to match the index mapping
The handler strips https:// from collection_endpoint automatically before building the client
IAM credentials are resolved via boto3.Session() — the Lambda execution role must have AOSS permissions in both IAM policy and the collection data access policy
